# 基于多模态大语言模型代理的太阳活动区自动分析系统

## 项目概述

本项目旨在构建一个基于多模态大语言模型的太阳活动区自动分析智能体系统。系统利用现成的多模态大语言模型（如 GPT-4V、LLaVA 等），实现太阳活动图像的自动识别、信息提取与结果整理，为用户提供规范化的太阳活动分析报告。

**项目目标：**
- 验证多模态大语言模型在太阳活动图像分析中的可行性
- 构建可展示的智能体原型系统
- 降低太阳活动分析的使用门槛

## 二、模块架构设计

### 2.1 用户交互层 (User Interaction Layer)

| 模块 | 功能描述 | 技术实现 |
|------|----------|----------|
| **图像上传模块** | 支持用户上传太阳观测图像（JPG/PNG/TIFF等格式） | Web表单/API接口 |
| **报告展示模块** | 展示分析结果报告，支持富文本显示 | HTML/CSS/JavaScript |
| **交互界面** | 提供用户友好的操作界面 | React/Vue/Streamlit |
| **历史查询模块** | 查询历史分析记录 | 数据库查询接口 |

### 2.2 智能体核心层 (Agent Core Layer)

| 模块 | 功能描述 | 关键技术 |
|------|----------|----------|
| **提示词引擎** | 生成结构化提示词，引导模型进行太阳活动分析 | Prompt Engineering |
| **任务拆解器** | 将复杂分析任务拆解为子任务序列 | 链式思考(Chain of Thought) |
| **结果整理器** | 将模型输出整理为规范报告格式 | 文本解析与格式化 |
| **工作流控制器** | 协调各模块执行顺序和数据流转 | 状态机/流程编排 |

### 2.3 多模态LLM接口层 (Multimodal LLM Interface)

| 组件 | 作用 | 选型建议 |
|------|------|----------|
| **图像理解接口** | 调用多模态模型进行图像识别 | GPT-4V / LLaVA / Qwen-VL |
| **文本生成接口** | 调用大语言模型生成分析报告 | GPT-4 / Claude 3 / Qwen |
| **智能体平台** | 提供工作流编排能力 | Coze / LangChain / LlamaIndex |

### 2.4 数据支撑层 (Data Support Layer)

| 模块 | 功能描述 | 存储方案 |
|------|----------|----------|
| **案例数据库** | 存储太阳活动图像案例及标注 | PostgreSQL + 图片存储服务 |
| **知识库** | 存储太阳物理学背景知识、术语定义 | 向量数据库(Pinecone/Weaviate) |
| **历史记录** | 存储用户上传记录和分析结果 | PostgreSQL |
| **配置管理** | 存储系统参数和模型配置 | 配置文件/数据库 |

## 四、核心功能设计

### 4.1 太阳活动图像识别

| 功能 | 描述 |
|------|------|
| **黑子区域识别** | 识别太阳表面的黑子位置、大小、形态 |
| **磁场特征分析** | 分析磁场强度、极性分布、磁类型 |
| **亮区变化检测** | 识别耀斑、日珥、暗条等亮区特征 |
| **活动风险评估** | 基于图像特征判断潜在活动风险等级 |

### 4.2 分析报告生成

| 功能 | 描述 |
|------|------|
| **现象描述** | 对图像中观测到的太阳活动现象进行自然语言描述 |
| **关键信息提取** | 提取黑子群编号、位置坐标、磁场参数等 |
| **初步判断** | 基于历史数据进行活动趋势判断 |
| **报告格式** | 结构化输出，包含摘要、详细分析、风险提示等 |

### 4.3 智能体工作流程

```
Step 1: 图像输入与预处理
        └── 格式校验、尺寸标准化、质量评估

Step 2: 提示词构建
        └── 注入太阳物理学背景知识
            定义分析任务框架
            设置输出格式约束

Step 3: 多模态模型调用
        └── 图像理解 + 文本生成
            可能的多次交互

Step 4: 结果解析与整理
        └── 提取关键信息
            验证输出格式
            格式化输出

Step 5: 报告生成与展示
        └── 结构化报告
            用户交互反馈
            历史记录保存
```

### 5.1 核心技术栈

## 六、API接口设计

### 6.1 接口列表

| 接口 | 方法 | 路径 | 功能 |
|------|------|------|------|
| 图像上传 | POST | `/api/upload` | 上传太阳图像文件 |
| 图像分析 | POST | `/api/analyze` | 执行图像分析任务 |
| 获取报告 | GET | `/api/report/{id}` | 获取指定分析报告 |
| 查询历史 | GET | `/api/history` | 查询历史分析记录 |
| 获取案例 | GET | `/api/cases` | 获取案例库列表 |

## 十、附录

### 10.1 太阳活动分析提示词模板

```
你是一位专业的太阳物理学家，请分析以下太阳图像。

图像信息：
- 数据源：{source}
- 波长：{wavelength}
- 时间：{timestamp}

请按照以下结构进行分析：

1. 黑子区域分析
   - 黑子群编号（如NOAA编号）
   - 位置坐标
   - 大小和形态描述
   - 磁场类型（alpha/beta/gamma/beta-gamma等）

2. 亮区特征分析
   - 耀斑活动（类型、强度、位置）
   - 日珥/暗条特征
   - 谱斑区域

3. 磁场特征分析
   - 磁场强度评估
   - 极性分布
   - 复杂性判断

4. 活动风险评估
   - 风险等级（低/中/高/极高）
   - 未来24-48小时活动预测
   - 对地影响评估

5. 总结与建议
   - 简明摘要
   - 监测建议

请使用专业但易懂的语言进行描述，输出格式为结构化JSON。
```
