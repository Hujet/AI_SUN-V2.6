# 研究架构：基于 Coze 智能体与 Helioviewer 的太阳活动区分析系统

## 1. 项目概况 (Project Overview)

*   **项目名称**：SolarInsight —— 零代码构建太阳黑子磁场形态分析助手

*   **研究目标**：
    1.  利用 Helioviewer 获取高质量的多波段太阳图像。
    2.  在 Coze 平台上通过提示词工程（Prompt Engineering）构建一个能“看懂”太阳磁图的 AI 智能体。
    3.  实现对太阳黑子磁场类型（Hale 分类）的自动化初步研判。

## 2. 数据获取策略：玩转 Helioviewer (Data Acquisition Strategy)

由于不使用编程抓取数据，我们将利用 **Helioviewer.org** 强大的可视化交互功能来构建我们的“训练与测试数据集”。根据你提供的网页信息，操作流程如下：

### 2.1 确定观测时间 (Observation Date)
*   **利用事件时间轴**：查看网页底部的 **"HEK Events Timeline"** (Heliophysics Events Knowledgebase)。寻找标记密集的日期，这通常意味着当天有耀斑或活跃的黑子群。
*   **时间跳转**：在左侧菜单的 **"Observation Date"** 栏目中，输入具体的日期和时间（UTC），点击 "Jump" 跳转到该时刻。

### 2.2 图层叠加与选择 (Layers & Image Selection)
为了让 AI 进行多模态分析，我们需要获取两类图像：
1.  **磁图 (Magnetogram)**：
    *   在 **"Images"** -> **"Data Sources"** 中选择 **SDO** -> **HMI** -> **Magnetogram**。
    *   *作用*：这是核心数据，用于看黑（负极）白（正极）斑块的分布。
2.  **极紫外图 (EUV)**：
    *   添加图层，选择 **SDO** -> **AIA** -> **171** 或 **193** 波段。
    *   *作用*：用于辅助确认活动区的位置和日冕环结构。

### 2.3 图像截取与保存 (Generate a Screenshot)
利用网页左侧的 **"Generate a Screenshot"** 功能：
*   **全日面图**：选择 "Full Viewport" 获取整个太阳的图像。
*   **局部特写**：选择 "Select Area"，用鼠标框选具体的活动区（黑子群），获取高分辨率局部图。
*   **保存**：点击生成后，下载图片。建议文件名格式为：`日期_活动区编号_波段.jpg` (例如 `20240510_AR3664_HMI.jpg`)。

## 3. Coze 智能体构建方案 (Coze Agent Construction)

在 Coze 平台上，我们将通过“人设”和“知识库”来替代代码逻辑。

### 3.1 机器人人设 (Persona & System Prompt)
在 Coze 的编排页面，输入以下系统提示词（System Prompt），这是机器人的“大脑”：

```markdown
# Role
你是一位专业的太阳物理观测助手，擅长通过视觉分析太阳动力学天文台 (SDO) 的图像。

# Skills
1. **磁图识别**：你能识别 HMI 磁图中的黑白区域。你知道白色代表正磁极 (Positive)，黑色代表负磁极 (Negative)。
2. **分类推理**：基于 Hale 分类法，根据正负磁极的分布形态判断黑子类型：
   - Alpha: 单极性黑子。
   - Beta: 双极结构，正负极分开明显。
   - Beta-Gamma: 双极组，但正负极分界线不规则，互相交错。
   - Delta: 极端复杂，相反极性的本影被挤压在同一个半影内。

# Workflow
当用户发送一张图片时：
1. **[图像确认]**：首先确认这是否是一张太阳图像。
2. **[视觉描述]**：详细描述你看到的黑子群。例如：“图像中心有一个巨大的黑子群，左侧呈现大片白色区域，右侧紧邻黑色区域。”
3. **[逻辑判断]**：分析黑白区域的混合程度。是泾渭分明？还是像太极图一样扭曲纠缠？
4. **[结论输出]**：给出最可能的 Hale 分类，并提醒用户参考 NOAA 官方数据。

# Constraints
- 如果图片模糊，建议用户去 Helioviewer.org 使用 "Select Area" 功能截取高清局部图。
- 保持科学严谨的态度。
```

### 3.2 知识库增强 (Knowledge Base - RAG)
为了让 AI 更懂行，我们需要利用 Coze 的**知识库**功能：
1.  **资料准备**：整理一份关于“Hale 磁场分类法”的图文文档（Word 或 PDF），包含 Alpha, Beta, Gamma, Delta 四种类型的标准定义和典型示例图。
2.  **上传**：在 Coze 中创建知识库 `Solar_Physics_Wiki`，上传上述文档。
3.  **关联**：将该知识库关联到你的 Bot 上。这样，当 AI 拿不准时，它会先去查阅这份文档，再回答问题。

### 3.3 工作流设计 (Workflow )
如果你希望流程更标准化，可以使用 Coze 的 **Workflow** 功能（图形化拖拽）：
*   **Start 节点**：接收用户图片。
*   **LLM 节点 (视觉分析)**：输入图片，提示词设为“描述图中黑色和白色斑块的空间关系”。
*   **LLM 节点 (推理分类)**：输入上一步的描述，结合知识库，输出分类结果。
*   **End 节点**：输出最终报告。

## 4. 研究实施步骤 (Implementation Steps)

### 第一阶段：数据准备 (Data Prep)
*   访问 **Helioviewer.org**。
*   利用 **"Observation Date"** 回溯过去几年的重大爆发事件（如 2024年5月的超级风暴）。
*   使用 **"Generate a Screenshot"** -> **"Select Area"** 功能，截取 20 张不同形态的黑子磁图（HMI Magnetogram）。
*   手动记录每张图对应的官方分类（可在 NOAA 网站查询历史记录，作为标准答案）。

### 第二阶段：Bot 调试与迭代 (Debugging)
*   **初次测试**：将截取的图片上传给 Coze Bot。
*   **评估**：
    *   AI 能分清黑色和白色吗？
    *   AI 能识别出“双极结构”吗？
*   **提示词优化 (Prompt Tuning)**：
    *   如果 AI 总是看错，尝试在提示词中加入：“请注意，灰色背景代表平静区，只有深黑和亮白才代表强磁场。”

### 第三阶段：多模态验证 (Multi-modal Verification)
*   在 Helioviewer 中，对同一个活动区，分别截取 **HMI 磁图** 和 **AIA 193 极紫外图**。
*   在 Coze 中一次性上传这两张图。
*   提问：“请对比这两张图，磁场最复杂的区域（磁图中的混乱区）是否对应着极紫外图中最亮的区域？”
*   *目的*：验证 AI 是否能建立“磁场-辐射”的物理关联。
